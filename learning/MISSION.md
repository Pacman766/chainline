# Mission: React + Next.js (App Router) + Payload CMS

## Why
Сменить рабочий стек с ExtJS/Java на React/Next.js. Цель — уверенно брать
продакшн-задачи на этом стеке (компоненты, страницы, фетчинг данных, headless CMS),
а не просто «читать чужой код». Этот проект (`payload-start`) — рабочий полигон.

## Success looks like
- Понимаю и объясняю модель рендеринга React (что и когда ре-рендерится) — не на уровне «работает магия».
- Чётко различаю Server vs Client Components в Next App Router и знаю, ПОЧЕМУ выбран тот или иной.
- Могу самостоятельно добавить новую страницу + фичу в этот проект: фетч данных из Payload, рендер, интерактивность.
- Понимаю модель данных Payload: коллекции, поля, access control, hooks, Local API.
- Могу пройти техническое собеседование mid-уровня по React/Next без «плавания» в фундаменте.

## Constraints
- Темп: интенсив (несколько часов в день), горизонт — недели, не месяцы.
- Формат: сначала концепция (схема/модель), потом практика на реальном коде проекта.
- Фон: сильный general web + ExtJS + немного Java. Не новичок в программировании —
  не нужно объяснять, что такое функция, замыкание, async/await.
- Язык объяснений — русский, код — английский.

## Out of scope (пока)
- Деплой/инфра (Vercel, CI/CD) — позже.
- Глубокий Tailwind/CSS — стили в проекте уже есть, не фокус.
- Стейт-менеджеры (Redux/Zustand) — в проекте Context, этого достаточно для старта.
- Тесты (Vitest/Playwright) — отдельный блок ближе к концу.
