# Практика 1 завершена — серверная страница /categories

Самостоятельно построил Server Component `/categories` (фетч из Payload Local API, рендер).

**Что показал:**
- Понял двухуровневый паттерн «список → счётчик на каждый элемент» (отдельный
  запрос к `products` на каждую категорию). Изначальная ошибка — попытка сделать всё
  одним запросом, спутав поля категории и товара (`_status` есть только у products).
- Верно объяснил, ПОЧЕМУ `Promise.all` вместо `for...of` с await: независимые запросы
  идут параллельно, время ≈ самый медленный, а не сумма.
- Применил нюансы из фидбэка: убрал лишний фрагмент, сам добавил `export const revalidate = 0`
  (осознанный выбор кеширования — мост в тему рендеринга/кеша Next).
- `key={cat.id}` — стабильный ключ, не индекс.

**Оформление** страницы сделал агент (по запросу) через frontend-design — вписал в
существующий дизайн-язык проекта (catalog-header, 1px-grid, акцент-линии). Это НЕ навык
учащегося, а showcase-полировка.

**Implications:** RSC-модель закреплена на практике. Готов к следующей теории —
routing в App Router (layouts, route groups `(frontend)`/`(payload)`, dynamic `[id]`,
вложенные layout'ы). Параллельно назрела тема кеширования/`revalidate` — всплыла дважды,
учащийся уже её касается руками.
