Серверная страница «Категории»

Своими руками создашь новый маршрут — чистый Server Component, который тянет данные из Payload и рендерит их. Никакого 'use client'.

Цель. Новая страница /categories: список всех категорий, у каждой — число опубликованных товаров и ссылка на отфильтрованный каталог. Всё на сервере, ноль клиентского JS.

Что построить

Страница по адресу http://localhost:3000/categories, которая:

  1. Достаёт все категории через Payload Local API.
  2. Для каждой категории считает, сколько в ней опубликованных товаров.
  3. Рендерит список: название категории, счётчик, и ссылку /products?category=<slug> (этот фильтр уже работает в существующей products/page.tsx).

Шаги

  1. Создай файл src/app/(frontend)/categories/page.tsx. Папка = маршрут, page.tsx = сама страница (это App Router из глоссария).
  2. Сделай export default async function — компонент должен быть async, потому что ждёт данные.
  3. Получи payload: const payload = await getPayload({ config }).
  4. Достань категории: payload.find({ collection: 'categories', limit: 100 }).
  5. Для счётчика товаров используй payload.count(...) с фильтром where (см. подсказку).
  6. Верни JSX со списком. Для ссылок — import Link from 'next/link'.
Правило проверки. Если в файле НЕТ 'use client' и НЕТ useState/onClick — ты всё сделал правильно как Server Component. Появилось желание добавить 'use client'? Остановись и спроси себя: где здесь интерактивность? (Её нет — значит не нужно.)

Критерии приёмки

  • /categories открывается без ошибок в браузере.
  • Видно все категории из БД, у каждой — корректное число опубликованных товаров.
  • Клик по категории ведёт на /products?category=<slug> и фильтрует каталог.
  • В файле нет 'use client' и нет React-хуков.
  • Данные приходят из Payload (ты НЕ хардкодил категории в массив).

Подсказки (открывай только если застрял)

💡 Как посчитать товары в категории

Payload Local API умеет считать без выгрузки документов:

const { totalDocs } = await payload.count({
  collection: 'products',
  where: {
    category: { equals: cat.id },
    _status:  { equals: 'published' },
  },
});

Категории нужно перебрать — а внутри map нельзя await. Поэтому собери промисы и дождись их через Promise.all (см. следующую подсказку).

💡 Перебор с await: Promise.all 
const categories = result.docs;
const withCounts = await Promise.all(
  categories.map(async (cat) => {
    const { totalDocs } = await payload.count({
      collection: 'products',
      where: { category: { equals: cat.id }, _status: { equals: 'published' } },
    });
    return { ...cat, count: totalDocs };
  })
);

🆘 Полное решение (загляни в последнюю очередь) 
import config from '@payload-config';
import { getPayload } from 'payload';
import Link from 'next/link';

export const revalidate = 0;

export default async function CategoriesPage() {
  const payload = await getPayload({ config });
  const { docs: categories } = await payload.find({
    collection: 'categories', limit: 100, sort: 'name',
  });

  const withCounts = await Promise.all(
    categories.map(async (cat) => {
      const { totalDocs } = await payload.count({
        collection: 'products',
        where: { category: { equals: cat.id }, _status: { equals: 'published' } },
      });
      return { ...cat, count: totalDocs };
    })
  );

  return (
    <div>
      <h1>Категории</h1>
      <ul>
        {withCounts.map((cat) => (
          <li key={cat.id}>
            <Link href={`/products?category=${cat.slug}`}>
              {cat.name} — {cat.count} товаров
            </Link>
          </li>
        ))}
      </ul>
    </div>
  );
}

Как проверить

  1. Запусти дев-сервер: npm run dev (если порт 3000 занят — освободи, см. твои правила).
  2. Открой http://localhost:3000/categories.
  3. Если категорий/товаров в БД нет — сначала засей: npx tsx src/seed.ts.
  4. Проверь счётчики и кликни по категории — должен открыться отфильтрованный каталог.
