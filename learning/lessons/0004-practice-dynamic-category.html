Динамический роут /categories/[slug]

Превратишь карточку категории из ссылки-фильтра в полноценную страницу раздела. Сам создашь динамический сегмент [slug], прочитаешь params и отдашь 404 на несуществующий slug.

Цель. Страница /categories/<slug> (например /categories/mountain): заголовок категории + сетка её опубликованных товаров. Несуществующий slug → честный 404.
Связь с прошлым. Список ты уже сделал в Практике 1 (/categories). Сейчас — «лист» этого дерева: страница одного раздела. Тот же двухуровневый паттерн (найти категорию → найти её товары), но категория теперь приходит из URL, а не из цикла.

Что построить

Файл src/app/(frontend)/categories/[slug]/page.tsx, который:

  1. Читает slug из params (помни: в Next 15 params — это Promise, его надо await).
  2. Находит категорию по этому slug через Payload Local API.
  3. Если категории нет — вызывает notFound() (отдаёт 404, а не падает с ошибкой).
  4. Находит опубликованные товары этой категории (тот же where, что в Практике 1).
  5. Рендерит заголовок категории + сетку товаров.

Шаги

  1. Создай папку categories/[slug]/ и в ней page.tsx. Квадратные скобки = динамический сегмент (Урок 3, раздел 4).
  2. Сигнатуру подсмотри в products/[id]/page.tsx:11 — там ровно тот же приём { params }: { params: Promise<{ id: string }> }, только у тебя slug.
  3. Категорию ищи через payload.find({ collection: 'categories', where: { slug: { equals: slug } }, limit: 1 }) — точь-в-точь как уже делают в products/page.tsx:54.
  4. Товары — where: { category: { equals: cat.id }, _status: { equals: 'published' } } (твой код из Практики 1).
  5. Для вёрстки можешь переиспользовать классы catalog-header и product-grid / product-card — они уже в styles.css. Картинки не обязательны, начни с названия + цены.
  6. В конце поменяй ссылку в карточках на /categories: пусть ведёт на /categories/${cat.slug} вместо /products?category=....
Две типичные ловушки. (1) notFound() импортируется из next/navigation и его надо вызвать (notFound()), а не вернуть. (2) params без await в Next 15 даст объект-Promise, и slug будет undefined — запрос молча вернёт пусто.

Критерии приёмки

Тайтовый feedback-loop — проверяешь сам в браузере:

  • /categories/<реальный-slug> показывает заголовок и товары этого раздела.
  • Товары совпадают с тем, что показывает /products?category=<тот-же-slug>.
  • /categories/qwerty-no-such отдаёт страницу 404, а не белый экран с ошибкой.
  • Клик по карточке на /categories ведёт на новую страницу раздела.
  • Никакого 'use client' — это чистый Server Component.
Подсказка 1 — где взять реальные slug'и для проверки

Загляни в админку /admin → Categories, либо открой /categories и наведись на карточку — slug виден в URL ссылки. В сид-данных это, например, mountain, road и т.д. (смотри src/seed-data).
