# Что стоит изучить дальше

[[000 Index|← Назад к карте]] | [[Проект — Что изучено|← Что изучено]]

> [!warning] Это не значит что ты не знаешь — это следующие шаги роста

## Payload CMS — продвинутые темы

- [ ] **Versions & Drafts** — черновики и история версий контента
- [ ] **Blocks field** — гибкий page builder (массив разных блоков)
- [ ] **Tabs** — группировка полей в вкладки в admin UI
- [ ] **Virtual fields** — вычисляемые поля (не хранятся в БД)
- [ ] **Plugins** — создание собственных плагинов Payload
- [ ] **Custom Admin UI** — кастомные компоненты в админ-панели
- [ ] **Migrations** — управление схемой БД при изменениях
- [ ] **Point field** — геолокация
- [ ] **Job Queue** — фоновые задачи (появилось в v3)

## Next.js — что не использовано

- [x] **ISR + revalidatePath** — мгновенное обновление страниц по webhook от CMS
- [x] **Parallel Routes** — `@slot` для сложных лейаутов
- [ ] **Intercepting Routes** — модальные окна через роутер
- [ ] **Server Actions** — формы без API роутов
- [ ] **Metadata API** — SEO через `generateMetadata()`
- [ ] **next/image** — оптимизация изображений (используешь Media, но не next/image)
- [ ] **Middleware** — защита роутов на уровне edge

## Производительность и деплой

- [ ] **ISR для каталога** — `/products` сейчас SSR, можно ISR с revalidate
- [ ] **Кеширование запросов** — `unstable_cache` для Payload запросов
- [ ] **Деплой Payload** — Railway, Render, VPS (SQLite → PostgreSQL для прода)
- [ ] **PostgreSQL adapter** — для production (SQLite не подходит для прода)

## Безопасность

- [ ] **Rate limiting** — на `/api/checkout` и `/api/auth/customer-login`
- [ ] **CORS настройка** — в payload.config.ts
- [ ] **Webhook secret validation** — уже есть паттерн, но стоит изучить глубже

## Приоритет изучения

```
1. ISR + revalidatePath  ← сразу применимо к проекту
2. Server Actions        ← упростит формы login/register
3. Versions & Drafts     ← стандартная фича CMS
4. PostgreSQL adapter    ← нужен перед деплоем
5. Blocks field          ← для page builder
```
